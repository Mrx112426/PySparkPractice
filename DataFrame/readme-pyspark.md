## Описание проекта

Учебный проект в формате Jupyter/Colab‑ноутбука, в котором на практике решаются несколько независимых задач аналитики данных с использованием **PySpark** и API DataFrame:

- анализ активности онлайн‑пользователей;
- анализ финансовых показателей ПАО «АвтоВАЗ» по годам;
- анализ данных вселенной *Star Wars* (виды, персонажи, организации).

Проект демонстрирует работу с комплексными типами данных (map, array), оконными функциями, агрегациями, joins, категоризацией по условиям и предварительной подготовкой данных (очистка и нормализация схемы).

Этот ноутбук можно позиционировать в портфолио как пример практического опыта работы с распределённой обработкой данных и аналитическими задачами на PySpark.

## Стек технологий

- **Язык:** Python 3  
- **Фреймворк для распределённой обработки:** Apache Spark (PySpark)  
- **Инструменты:**  
  - SparkSession, PySpark SQL, функции `pyspark.sql.functions`  
  - Оконные функции (`Window`)  
  - Работа с `StructType`, `ArrayType`, `MapType`  
- **Среда выполнения:** Jupyter / Google Colab  

## Структура проекта

Проект реализован в одном Jupyter/Colab‑ноутбуке и логически разделён на три независимых блока:

1. **Анализ активности онлайн‑пользователей**  
2. **Анализ финансовых данных ПАО «АвтоВАЗ»**  
3. **Анализ данных вселенной Star Wars**  

Каждый блок создаёт собственный `SparkSession` (с уникальным `appName`), загружает свои данные и решает отдельный набор аналитических задач.

## Ключевые задачи и реализованные решения

### 1. Анализ активности онлайн‑пользователей

**Цель:** смоделировать логи пользовательской активности и рассчитать ключевые метрики по сессиям и посещённым страницам.

Основные моменты:

- Создание DataFrame `df_user_activity` со схемой:
  - `user_id` — идентификатор пользователя;
  - `date` — дата активности;
  - `device_type` — карта вида `{"mobile": int, "desktop": int, "tablet": int}` с количеством сессий по устройствам;
  - `page_views` — массив посещённых страниц;
  - `rating` — оценка удобства (usability rating).
- Вывод схемы DataFrame и первичного среза данных.
- Обогащение данных:
  - вычисление числа мобильных сессий `mobile_sessions` c безопасной обработкой отсутствующих ключей (`coalesce`);
  - расчёт общего количества сессий по всем устройствам `total_sessions_count`.
- Агрегация по пользователям:
  - `total_sessions_all_time` — суммарное количество сессий за весь период;
  - `unique_visited_pages_all_time` — список уникальных посещённых страниц для каждого пользователя (через `collect_set` + `flatten` + `array_distinct`);
  - сортировка пользователей по убыванию общего числа сессий.
- Фильтрация:
  - выбор только записей с `rating > 3.5`;
  - сортировка по оценке удобства в порядке убывания.

**Что демонстрирует:**

- работу с map‑типами и массивами в PySpark;
- добавление вычисляемых колонок и агрегации по пользователям;
- базовый поведенческий анализ пользователей по устройствам и страницам.

### 2. Анализ финансовых данных ПАО «АвтоВАЗ»

**Цель:** проанализировать динамику финансовых показателей компании по годам, используя оконные функции и категоризацию по уровням прибыли.

Основные моменты:

- Загрузка CSV‑файла `avtovaz.csv` c включёнными опциями заголовка и автоопределения типов.
- Предварительная обработка:
  - нормализация названий столбцов (нижний регистр, замена пробелов/точек на подчёркивания);
  - использование числовых типов для финансовых показателей.
- Задание 1: динамика чистой прибыли и ранжирование по росту:
  - использование оконной спецификации `Window.orderBy('год')`;
  - вычисление чистой прибыли за предыдущий год через `lag`;
  - расчёт изменения прибыли `изменение_прибыли` по сравнению с предыдущим годом;
  - ранжирование годов по **абсолютной** величине изменения прибыли (`rank` + `abs`);
  - вывод таблицы с годом, прибылью текущего и предыдущего года, изменением и рангом.
- Задание 2: категоризация годов по уровням прибыли:
  - создание категории `категория_прибыли` через каскад `when` с классами:
    - «Высокоприбыльный год» (> 5000 млн руб.)
    - «Среднеприбыльный год» (1000–5000 млн руб.)
    - «Низкоприбыльный год» (0–1000 млн руб.)
    - «Убыточный год» (< 0)
    - «Нулевая прибыль» (= 0)
  - агрегация по категориям:
    - `количество_лет` — число лет в категории;
    - `средняя_общая_реализация_тыс_шт` — средняя сумма внутренней и внешней реализации (в тыс. штук), с округлением до двух знаков;
  - сортировка категорий по количеству лет по убыванию.

**Что демонстрирует:**

- использование оконных функций (`lag`, `rank`) для анализа динамики;
- построение витрины с изменением прибыли и её ранжированием;
- категоризацию периодов по бизнес‑правилам и последующую агрегацию.

### 3. Анализ данных вселенной Star Wars

**Цель:** на основе нескольких датасетов в формате Parquet провести анализ видов, персонажей и организаций вселенной *Star Wars*.

Основные моменты:

- Загрузка трёх Parquet‑датасетов:
  - `species.parquet` — справочник видов (`df_zv_species`),
  - `organizations.parquet` — организации (`df_zv_organizations`),
  - `characters.parquet` — персонажи (`df_zv_characters`).
- Удаление дубликатов по ключевым полям для повышения качества данных.

Реализованные задачи:

1. **Распространённость видов и классификаций**
   - Подсчёт количества персонажей по каждому виду (`species`) и сортировка по убыванию (топ‑10 видов).
   - Join видов и персонажей по `species == name` и подсчёт количества персонажей по биологическим классификациям (`classification`).

2. **Средний рост по классификациям**
   - Группировка по `classification` в `df_zv_species` и расчёт среднего роста `average_height` с округлением до одного знака.

3. **Члены Ордена Джедаев и Ордена Ситхов**
   - Преобразование строковых полей `leader` и `members` в массивы через `split`.
   - Разворачивание массивов в строки (`explode_outer`) и фильтрация по организациям `Jedi Order` и `Sith Order`.
   - Формирование общего списка «организация – участник» с лидерами и членами.

4. **Топ‑5 старейших и самых юных персонажей**
   - Фильтрация персонажей с ненулевым `year_born`.
   - Сортировка по году рождения по убыванию (старейшие) и по возрастанию (самые юные).
   - Вывод топ‑5 в каждой группе.

5. **Персонажи с экстремальным индексом массы тела (ИМТ)**
   - Расчёт ИМТ как `weight / height**2` с округлением до двух знаков.
   - Отбор 5 персонажей с максимальным и 5 с минимальным ИМТ (без NULL).

**Что демонстрирует:**

- работу с Parquet‑файлами и удалением дубликатов;
- join справочников и факт‑таблиц;
- преобразование строковых списков в массивы и их разворачивание;
- расчёт производных показателей и ранжирование.
