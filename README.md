# PySpark RDD Analytics: Многодоменная обработка данных

**Комплексный pet-проект** по распределённой обработке данных с использованием **Apache Spark RDD API** в Google Colab. Ноутбук демонстрирует практическое применение PySpark на пяти различных доменах данных с production-ready техниками аналитики.

---

## 📋 Описание проекта

Этот проект предоставляет практические примеры **RDD трансформаций и действий** на разнообразных датасетах:
- Логи промышленных IoT-сенсоров с обнаружением аномалий
- Анализ безопасности паролей с извлечением паттернов
- Лингвистический анализ русского словаря
- Парсинг логов веб-сервера и HTTP-аналитика
- Метрики вовлечённости постов в соцсетях
- Геополитический анализ данных (датасет стран)

**Ключевой фокус:** Распределённая обработка данных, очистка данных, ad-hoc аналитика и оптимизация производительности в Spark.

## 🔧 RDD методы и использованные техники

### **Основные трансформации**
```python
.map()           # Извлечение полей, преобразование типов, feature engineering
.filter()        # Условная фильтрация, обнаружение аномалий
.reduceByKey()   # Агрегация, группировка, подсчёт
.sortByKey()     # Сортировка по ключу для упорядоченного вывода
.distinct()      # Удаление дубликатов (уникальные IP-адреса и т.д.)
```

### **Действия (Actions)**
```python
.collect()       # Материализация результатов на драйвер
.count()         # Общее количество записей
.mean()          # Среднее значение
.max() / .min()  # Экстремальные значения
.sum()           # Суммирование
.top(n)          # N наибольших элементов
.takeOrdered()   # Отсортированные N элементов
.countByValue()  # Подсчёт частот (на драйвере)
```

### **Паттерны производительности**
- **Распределённая агрегация:** Использование `reduceByKey()` для больших датасетов (пароли) вместо `countByValue()`
- **Ленивые вычисления:** Трансформации не выполняются до вызова действия
- **Обработка NULL:** Явная проверка типов перед приведением (`if x != "NULL"`)
- **Regex парсинг:** Apache логи: `r'(\S+) - - \[(.*?)\] "(.*?)" (\d{3}) (\d+)'`

## 📈 Архитектура кода

### **Инициализация сессии**
```python
spark = SparkSession.builder \
    .appName("reduceByKey_Example") \
    .master("local[*]") \
    .getOrCreate()
sc = spark.sparkContext
```

### **Паттерны загрузки данных**
- **В памяти:** `sc.parallelize()` для строковых данных (логи сенсоров)
- **Из файлов:** `sc.textFile()` для внешних файлов (пароли, словарь, логи)

### **Стратегия парсинга**
Каждый датасет включает пользовательскую логику парсинга:
```python
# Логи сенсоров: Разделитель | с приведением типов
parsed_rdd = field_logs.map(lambda f: (
    f[0], f[1], float(f[2]) if f[2] != "NULL" else None, ...
))

# Веб-логи: Regex извлечение для Apache формата
pattern = r'(\S+) - - \[(.*?)\] "(.*?)" (\d{3}) (\d+)'

# CSV данные: Разделение запятой с фильтром заголовков
posts_no_header_rdd = posts_rdd.filter(lambda x: x != header)
```

## 📊 Примечания по производительности

- **Масштаб:** Демонстрация на датасетах от 39 строк (посты) до 681K+ записей (пароли)
- **Локальное выполнение:** Все вычисления выполняются на одной машине с режимом `local[*]`
- **Оптимизация:** `reduceByKey()` используется для распределённой агрегации во избежание OOM на драйвере
- **Производительность Regex:** Pre-compiled паттерны для повторяемого парсинга логов
